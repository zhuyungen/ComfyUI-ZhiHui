# V2.1 更新说明 - 模型路径优化

## 🎯 本次更新内容

### ✅ 1. 图片反推新增遮罩输出

为图片反推节点添加遮罩（MASK）输出，便于与其他节点连接使用。

#### 新增输出
**🎨 智绘_图片反推** 节点现在输出三个值：
1. **原图(透传)** - IMAGE 类型
2. **描述文本** - STRING 类型
3. **遮罩(全白)** - MASK 类型 ⭐ 新增

#### 遮罩说明
- **类型**: MASK，形状为 `(batch_size, height, width)`
- **值**: 全白遮罩，所有像素值为 `1.0`
- **含义**: 表示整张图片都是有效区域
- **用途**: 可连接到其他需要遮罩的节点（如图像合成、混合等）

#### 使用场景
```
🎨 智绘_图片反推
  ├─ 原图 → 后续处理
  ├─ 描述文本 → 文本编码/显示
  └─ 遮罩 → 图像合成节点
```

### ✅ 2. 智能抠图模型路径优化

将抠图模型的存储位置从系统默认路径改为 ComfyUI 统一管理目录。

### ✅ 3. 预设文本管理增强 📝

为预设文本功能增加文本输入、组合和专门的保存节点。

#### 加载节点增强
**🎨 智绘_预设文本加载** 节点现在支持：
1. **自定义文本输入** - 多行文本框
2. **组合模式** - 4种模式可选
3. **分隔符设置** - 自定义分隔符

#### 保存节点 💾 新增
**💾 智绘_保存预设** - 专门的预设保存节点：
- **独立职责** - 专注于保存功能
- **明确操作** - 清晰的保存流程
- **状态反馈** - 实时显示保存结果
- **文本透传** - 可继续连接其他节点

#### 组合模式
- **仅使用预设** - 传统模式
- **仅使用输入文本** - 完全自定义
- **预设+输入文本** - 预设在前
- **输入文本+预设** - 自定义在前

#### 使用场景
```
🎨 智绘_预设文本加载
  ├─ preset: "画质词"
  ├─ custom_text: "主体描述"
  ├─ mode: "输入文本+预设"
  └─ separator: ", "
      ↓ 输出文本
💾 智绘_保存预设
  ├─ text: (连接上游)
  └─ preset_name: "自定义预设"
      ↓
    保存到 presets.json ✅
```

详细说明请查看：
- [预设文本加载增强说明.md](预设文本加载增强说明.md) - 组合功能
- [智绘预设保存-使用指南.md](智绘预设保存-使用指南.md) - 保存节点使用 ⭐

## 📁 路径变更对比

### 旧版本 (V2.0 及之前)
```
抠图模型路径: ~/.u2net
Windows: C:\Users\Administrator\.u2net
Linux/Mac: ~/.u2net
```

❌ **问题**:
- 与 ComfyUI 目录分离，不便管理
- Windows 用户路径混乱（C盘用户目录）
- 不符合 ComfyUI 生态规范

### 新版本 (V2.1)
```
优先路径: ComfyUI/models/rembg/
备用路径: ComfyUI/.cache/rembg/
回退路径: ~/.u2net (兜底)
```

✅ **优势**:
- 与 ComfyUI 模型统一管理
- 路径更清晰，便于查找
- 支持备份迁移
- 符合 ComfyUI 标准

## 🔧 实现机制

### 自动路径检测逻辑

```python
def get_rembg_cache_path():
    try:
        # 1. 定位 ComfyUI 根目录
        comfyui_root = 向上5层查找

        # 2. 优先使用 models 目录
        if models/ 存在:
            使用 ComfyUI/models/rembg/
        else:
            使用 ComfyUI/.cache/rembg/

        # 3. 设置环境变量
        os.environ['U2NET_HOME'] = 选定路径

    except:
        # 4. 回退到默认路径
        使用 ~/.u2net
```

### 环境变量设置
自动设置 `U2NET_HOME` 环境变量，确保 rembg 库使用正确路径。

### 对新用户
无需任何操作，直接使用即可。模型会自动下载到正确位置。

### 对已安装用户

#### 选项 1: 重新下载（推荐）
1. 删除旧模型：`rm -rf ~/.u2net`

**缺点**: 需要重新下载（~176MB）

```bash
# Windows
xcopy /E /I "C:\Users\Administrator\.u2net" "你的ComfyUI路径\models\rembg"

# Linux/Mac
cp -r ~/.u2net/* /path/to/ComfyUI/models/rembg/
```

**优点**: 无需重新下载
**缺点**: 需手动操作

#### 选项 3: 保持不变
如果不想迁移，旧路径仍然有效（作为回退方案）。

## 🔍 常见问题

### Q1: 升级后找不到之前的模型？
**A**: 插件会自动使用新路径。如果想继续使用旧模型，请参考"升级指南 - 选项2: 手动迁移"。

### Q2: 为什么有两个备选路径？
**A**:
- `models/rembg/` - 正式模型目录，便于管理和备份
- `.cache/rembg/` - 缓存目录，适合临时或自动管理

系统会自动选择最合适的路径。

### Q3: 可以自定义路径吗？
**A**: 可以，通过设置环境变量：
```bash
export U2NET_HOME=/custom/path
```
启动 ComfyUI 前设置即可。

### Q4: 旧模型会自动删除吗？
**A**: 不会。`~/.u2net` 目录需要手动删除。确认新路径工作正常后再删除。

### Q5: 路径检测失败怎么办？
**A**: 插件会自动回退到 `~/.u2net`，保证功能可用。如需帮助，请查看控制台错误信息。

## 🎨 功能一：图片反推（Image Captioning）V2.0

### 🆕 V2.0 更新内容
- ✅ **支持批量处理** - 可同时处理多张图片
- ✅ **中文模型名称** - 更清晰易懂的选项
- ✅ **透传原图** - 输出包含原图和描述文本
- ✅ **三种模式** - 自动生成、问答模式、引导生成
- ✅ **路径信息节点** - 查看模型缓存位置
- ✅ **遮罩输出** - 新增MASK输出便于节点连接 (V2.1)

### 节点列表
- **🎨 智绘_图片反推** - 主功能节点
- **📁 智绘_模型路径信息** - 查看模型存储位置

### 参数说明（图片反推节点）

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **images** | 输入图片（支持批量）| - |
| **model_name** | 模型选择 | `BLIP-Large (推荐-精度高)` |
| **mode** | 工作模式 | 见下表 |
| **prompt_or_question** | 提示词/问题 | 根据模式填写 |
| **max_length** | 最大描述长度 | 75 |
| **num_beams** | 束搜索数量 | 5 (3-5推荐) |
| **temperature** | 创造性温度 | 1.0 (标准) |

### 模型对比

| 模型 | 精度 | 速度 | 显存 | 推荐场景 |
|------|------|------|------|----------|
| **BLIP-Large (推荐-精度高)** | ⭐⭐⭐⭐⭐ | ⚡⚡⚡ | ~2GB | 通用推荐 |
| **BLIP-Base (快速-精度中)** | ⭐⭐⭐⭐ | ⚡⚡⚡⚡ | ~1GB | 快速处理 |
| **BLIP2-2.7B (最强-显存需3GB+)** | ⭐⭐⭐⭐⭐ | ⚡⚡ | ~5GB | 最高精度 |

### 工作模式说明

#### 1️⃣ 自动生成描述
- **适用场景**：快速分析图片内容
- **提示词框**：留空即可
- **输出示例**：`a woman sitting on a bench in a park`

#### 2️⃣ 问答模式
- **适用场景**：针对图片提问
- **提示词框**：输入英文问题
  - `What is in the image?`
  - `What color is the sky?`
  - `How many people are there?`
- **输出示例**：`three people`

#### 3️⃣ 引导生成
- **适用场景**：生成特定风格描述
- **提示词框**：输入开头文字
  - `a photography of`
  - `an artistic rendering of`
- **输出示例**：`a photography of a sunset over mountains`

### 输出说明
- **原图(透传)** - IMAGE 类型，可连接到其他节点
- **描述文本** - STRING 类型，多张图片用换行符分隔
- **遮罩(全白)** - MASK 类型，值为1.0的全白遮罩，表示整张图片有效区域

📖 详细遮罩使用说明请查看 [图片反推遮罩说明.md](图片反推遮罩说明.md)

### 模型缓存路径
- **Windows**: `C:\Users\用户名\.cache\huggingface\hub`
- **Linux/Mac**: `~/.cache/huggingface/hub`

使用 **📁 智绘_模型路径信息** 节点可查看详细信息。

### 智能抠图节点

#### 参数说明

| 参数 | 说明 | 推荐场景 |
|------|------|----------|
| **images** | 输入图片（支持批量）| - |
| **model** | 模型选择 | 见下表 |
| **alpha_matting** | 边缘优化开关 | 需要精细边缘时启用 |
| **foreground_threshold** | 前景阈值(0-255) | 240（边缘锐利）|
| **background_threshold** | 背景阈值(0-255) | 10（背景干净）|
| **post_process_mask** | 蒙版后处理 | 去除小噪点时启用 |

#### 模型选择指南

| 模型名称 | 实际模型 | 适用场景 | 速度 | 精度 |
|---------|----------|----------|------|------|
| **通用抠图-标准 (推荐)** | u2net | 通用场景，平衡速度和精度 | ⚡⚡⚡ | ⭐⭐⭐⭐ |
| **通用抠图-快速 (速度优先)** | u2netp | 快速批量处理 | ⚡⚡⚡⚡ | ⭐⭐⭐ |
| **人像抠图-专用 (人物)** | u2net_human_seg | 人像照片 | ⚡⚡⚡ | ⭐⭐⭐⭐⭐ |
| **人像抠图-高精度 (细节)** | silueta | 需要完美边缘的人像 | ⚡⚡ | ⭐⭐⭐⭐⭐ |

# 图片反推节点遮罩输出说明

## 🎯 更新内容

**🎨 智绘_图片反推** 节点现已支持遮罩输出！

---

## 📊 输出变化

### 之前（V2.0）
```
输出:
├─ 原图(透传) - IMAGE
└─ 描述文本 - STRING
```

### 现在（V2.1）
```
输出:
├─ 原图(透传) - IMAGE
├─ 描述文本 - STRING
└─ 遮罩(全白) - MASK ⭐ 新增
```

---

## 🔍 遮罩详细说明

### 基本信息
- **类型**: MASK
- **形状**: `(batch_size, height, width)`
- **数据类型**: torch.Tensor
- **值范围**: 全部为 `1.0` (全白)

### 含义
遮罩值为 `1.0` 表示该像素完全可见/有效。由于图片反推是对整张图片进行分析，因此输出全白遮罩表示整张图片都是有效区域。

### 技术实现
```python
# 生成全白遮罩
mask = torch.ones((images.shape[0], images.shape[1], images.shape[2]))
# 形状示例: (batch=4, height=512, width=512)
```

---

## 💡 使用场景

### 场景 1: 与抠图节点配合
```
Load Image
  → 🎨 智绘_图片反推
    ├─ 原图 → 🎨 智能抠图
    ├─ 遮罩 → (作为初始遮罩)
    └─ 描述文本 → Text Display
```

### 场景 2: 图像混合工作流
```
图片A → 🎨 智绘_图片反推
       ├─ 原图 ────┐
       └─ 遮罩 ────┤
                    ├→ Image Composite/Blend
图片B ──────────────┤
                    └→ 输出
```

### 场景 3: 条件处理流程
```
Load Image
  → 🎨 智绘_图片反推
    ├─ 原图 → Apply Processing
    ├─ 遮罩 → Mask Editor (可选修改)
    └─ 描述文本 → Condition Check
```

### 场景 4: 批量处理管道
```
Batch Load (100张)
  → 🎨 智绘_图片反推
    ├─ 原图 → Batch Processor
    ├─ 遮罩 → Quality Control Filter
    └─ 描述文本 → Tag Generator
```

## 🔧 代码示例

### 使用遮罩进行图像合成
```python
# 假设从图片反推节点获得
image = ...  # IMAGE tensor
caption = ...  # STRING
mask = ...  # MASK tensor (全白)

# 方式1: 直接使用
result = image * mask.unsqueeze(-1)  # 应用遮罩

# 方式2: 与其他遮罩组合
custom_mask = ...  # 自定义遮罩
combined_mask = mask * custom_mask  # 遮罩相乘
result = image * combined_mask.unsqueeze(-1)

# 方式3: 条件处理
if mask.mean() > 0.5:  # 检查遮罩有效性
    process_image(image)
```

## ❓ 常见问题

### Q1: 为什么是全白遮罩？
**A**: 图片反推（Image Captioning）是对整张图片进行分析，不涉及区域分割。全白遮罩表示整张图片都参与分析，这是最合理的表示。

### Q2: 可以修改遮罩吗？
**A**: 可以！输出的遮罩可以连接到 Mask Editor 等节点进行修改，或与其他遮罩组合使用。

### Q3: 遮罩占用多少内存？
**A**: 很少。对于 512x512 的图片批次（4张），遮罩大小为：
```
4 × 512 × 512 × 4 bytes = 4 MB
```

### Q4: 批量处理时遮罩如何？
**A**: 每张图片都有对应的遮罩。形状为 `(batch_size, height, width)`，每张图片独立。

### Q5: 可以生成部分遮罩吗？
**A**: 图片反推节点本身只生成全白遮罩。如需部分遮罩，请使用 **🎨 智能抠图** 节点。

## 🔧 参数说明

### 1. preset (预设选择)
- **类型**: 下拉菜单
- **选项**:
  - `(不使用预设)` - 不使用任何预设
  - 其他选项从 `presets.json` 加载
- **默认**: `(不使用预设)`

### 2. custom_text (自定义文本) ⭐ 新增
- **类型**: 多行文本框
- **用途**: 手动输入文本内容
- **支持**: 多行文本、UTF-8 编码
- **占位符**: "在此输入自定义文本，或选择上方预设"

- **类型**: 下拉菜单
- **选项**:
  - `仅使用预设` - 只输出预设内容（默认）
  - `仅使用输入文本` - 只输出自定义文本
  - `预设+输入文本` - 预设在前，自定义在后
  - `输入文本+预设` - 自定义在前，预设在后

### 4. separator (分隔符) ⭐ 新增
- **类型**: 单行文本
- **默认**: `\n` (换行符)
- **用途**: 组合时的分隔符
- **常用值**:
  - `\n` - 换行
  - `, ` - 逗号空格
  - ` ` - 空格
  - 自定义字符

**🍌 智绘香蕉2 (极速版)** 节点新增"智绘显示"输出！

---

## 📊 输出变化

### 旧版本
```
输出:
├─ image (生成图片)
└─ response_text (响应文本)
```

### 新版本 V2.1 ⭐
```
输出:
├─ 生成图片 (IMAGE)
├─ 详细信息 (STRING)
└─ 智绘显示 (STRING) ← 新增
```

---

## 🔍 输出详解

### 1. 生成图片 (IMAGE)
- **类型**: IMAGE
- **说明**: API 生成的图片
- **用途**: 连接到后续图像处理节点

### 2. 详细信息 (STRING)
- **类型**: STRING
- **内容**: 简洁的文本信息
- **格式**:
  ```
  Prompt: {提示词}
  Size: {尺寸} ({比例})
  ```
- **用途**: 快速查看基本信息

### 3. 智绘显示 (STRING) ⭐ 新增
- **类型**: STRING
- **内容**: 格式化的详细显示
- **特点**: 美观的表格布局
- **用途**: 查看完整生成信息

# ComfyUI-ZHIHUI 工具箱集成说明

## 🎯 集成概述

已成功将 Dapao-Toolbox 的核心功能集成到 ComfyUI-ZHIHUI 中，所有节点已改名为"智绘"系列，统一归类到 **智绘灵箱/工具箱** 分类下。

---

## 📦 新增节点列表

### 1. 🧠 智绘_显存内存优化
**原名**: 显存丨内存智能优化
**功能**: 智能管理显存和内存，优化 ComfyUI 性能

**主要特性**:
- ⚡ 预留显存设置
- 🧹 低内存/低显存时自动卸载模型
- 🧽 运行时清空缓存
- 🧯 强制垃圾回收（GC）
- 📊 实时显示内存/显存状态

**使用建议**:
- 放在工作流最前面
- 或在加载大模型/高分辨率采样前使用

**参数说明**:
- 预留显存GB: 0.6 (默认)
- 内存安全余量GB: 4.0 (默认)
- 显存安全余量GB: 0.0 (默认)

---

### 2. 🔢 智绘_多图片开关
**原名**: 多图片开关节点
**功能**: 在多个图片之间智能切换

**主要特性**:
- ✨ 智能动态输入（2-20个输入端口）
- 🎯 编号选择（1-20）
- ⏭️ 自动跳过空图片
- 🔄 循环模式
- 📊 详细信息输出

**输出参数**:
- 🖼️ 图像
- ℹ️ 信息
- 🔢 索引
- 📊 总数

**使用场景**:
- 批量图片处理时快速切换
- 测试不同图片的处理效果
- 工作流中选择多个输入之一

---

### 3. 📐 智绘_图片排列
**原名**: 图片排列节点
**功能**: 将基准图和批次图按指定方式排列

**主要特性**:
- 🎯 四种排列方向（左右/上下/左上/右上）
- 📁 支持文件夹批量加载
- 📊 三种布局模式（自动/固定列数/固定行数）
- 🎨 三种缩放模式（适应/裁剪/拉伸）
- 🌈 自定义背景颜色和边框

**参数说明**:
- 排列方向: 左右/上下/左上/右上
- 基准图尺寸模式: 默认/自定义最长边
- 布局模式: 自动/固定列数/固定行数
- 小图尺寸: 64-1024px
- 缩放模式: 适应/裁剪/拉伸

**使用场景**:
- 原图与处理结果对比展示
- 批量预览（主图配多张缩略图）
- 工作流步骤展示
- 效果对比

---

### 4. 📦 智绘_制作批次
**原名**: 制作图像批次
**功能**: 将多个单独图像合并成一个批次

**主要特性**:
- ✨ 智能动态输入（2-20个输入端口）
- 🔗 批次合并
- ⚡ 自动跳过空输入
- 📊 数量统计

**输出参数**:
- 🖼️ 图像批次
- 📊 图像数量

**使用场景**:
- 批量处理多张图像
- 动画生成（多帧合并）
- 对比测试
- 工作流优化

**技术说明**:
- 沿batch维度（第0维）连接
- 所有输入图像尺寸必须相同
- 输出可被批处理节点使用

---

### 5. 📁 智绘_文件夹加载
**原名**: 文件夹加载图像
**功能**: 从文件夹批量加载图片

**主要特性**:
- 📂 指定文件夹路径
- 🔢 限制加载数量
- 📊 批次输出
- 🖼️ 支持多种格式（jpg/png/bmp/webp/tiff）

**参数说明**:
- 文件夹路径: 图片所在目录
- 最大数量: 限制加载图片数量

**使用场景**:
- 批量处理文件夹图片
- 数据集加载
- 批量测试

---

### 6. 🎲 智绘_随机提示词提取
**原名**: 随机提示词行提取
**功能**: 从多行提示词中随机提取指定数量的行

**主要特性**:
- 🎲 随机抽取
- 🔢 指定数量
- 📝 多行文本支持
- 🔄 可重复性（通过seed）

**参数说明**:
- 输入文本: 多行提示词（每行一个）
- 提取数量: 要随机选择的行数
- 种子: 随机种子（可选）

**使用场景**:
- 随机化提示词组合
- 生成多样化结果
- 测试不同提示词效果

---

### 7. 🎲 智绘_随机提示词组合
**原名**: 随机提示词行组合
**功能**: 将多组提示词随机组合

**主要特性**:
- 🎲 多组随机组合
- 🔗 自定义分隔符
- 📝 灵活输入
- 🔄 可重复性（通过seed）

**参数说明**:
- 多组输入文本
- 分隔符: 默认逗号或自定义
- 种子: 随机种子（可选）

**使用场景**:
- 组合不同类型的提示词
- 生成随机但结构化的提示词
- 批量测试提示词组合效果

# 智绘_保存PSD 功能说明

## 🎯 功能概述

**💾 智绘_保存PSD** 节点可以将多个图像保存为 PSD（Photoshop Document）格式文件，每个图像会自动转换为单独的图层。

---

## ✨ 功能特点

- ✅ **批量图像转PSD**: 支持单张或多张图像批量处理
- ✅ **自动图层管理**: 每个图像自动成为独立图层（图层 1, 图层 2...）
- ✅ **智能居中对齐**: 图层自动居中对齐在画布上
- ✅ **自定义保存路径**: 支持指定保存目录
- ✅ **灵活文件命名**: 支持自定义文件名前缀
- ✅ **自适应画布**: 自动计算画布尺寸（取所有图像的最大宽高）
- ✅ **路径错误修复**: 已修复相对路径与绝对路径混合错误 ⭐

---

## ⚙️ 技术细节

### 图层处理机制
- **格式转换**: 所有图像自动转换为 RGBA 格式
- **居中对齐**: 图层自动居中对齐在画布上
- **画布计算**: 画布尺寸 = max(所有图像宽度) × max(所有图像高度)
- **图层命名**: 按顺序命名为"图层 1"、"图层 2"等

### 文件命名规则
```
格式: {文件名前缀}_{计数器:05d}_.psd
示例: zhihui_psd_00001_.psd, my_project_00002_.psd
```

### 压缩方式
- 使用 `Compression.raw`（无压缩）
- 确保最大兼容性
- 避免某些 pytoshop 版本的 packbits 压缩 bug

### 🔧 路径处理（V2.1+ 修复）
```python
# 修复前：可能导致路径混合错误
base_output_dir = custom_path

# 修复后：自动转换为绝对路径
abs_custom_path = os.path.abspath(custom_path)
base_output_dir = abs_custom_path
```

- 大尺寸图像建议先压缩再保存

## ⚠️ 注意事项

### 1. 依赖要求
- **必须安装**: pytoshop 库
- **错误提示**: `智绘_保存PSD: pytoshop 模块未找到`
- **解决方法**: `pip install pytoshop`

### 2. 内存占用
- PSD 文件包含所有图层数据，文件较大
- 大批量图像可能占用较多内存
- 建议分批处理大量图像

### 3. 图像尺寸
- 不同尺寸的图像会自动居中对齐
- 画布尺寸取所有图像的最大宽高
- 小图像会在大画布中居中显示

### 4. 兼容性
- 生成的PSD文件可在Photoshop、GIMP等软件中打开
- 所有图层均为普通图像图层（非智能对象）
- 支持透明度信息保存

### 5. 路径安全 ⭐ 已修复
- 自动处理相对路径与绝对路径混合问题
- 路径创建失败时自动回退到默认路径
- 支持中文路径名称

## 🐛 错误处理

### 错误：pytoshop 模块未找到
**解决方法**:
```bash
pip install pytoshop
```

### 错误：自定义路径无法创建
**原因**: 路径权限不足或格式错误
**解决方法**:
- 检查路径格式（Windows: `D:/path`, Linux: `/path/to/dir`）
- 确保有写入权限
- 节点会自动回退到默认输出路径

### 错误：Can't mix absolute and relative paths ⭐ 已修复
**原因**: 相对路径与绝对路径混合使用
**解决方法**: 已在V2.1+版本中自动修复，无需手动处理

### 错误：保存 PSD 文件时出错
**可能原因**:
- 磁盘空间不足
- 文件路径过长
- 权限问题

**解决方法**:
- 检查磁盘空间
- 使用较短的文件名前缀
- 检查目录权限
